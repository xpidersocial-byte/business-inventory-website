System Documentation

Technical specifications and architectural overview of the FBIHM System v2.5

1. Overview 2. Database Schema 3. Core Algorithms 4. PWA & Notifications 5. API Reference

1. System Overview

FBIHM is a full-stack inventory management system built with Flask (Python) and MongoDB. It features a modern PWA frontend with real-time notifications and AI-driven business intelligence.

  • Backend: Python 3.13, Flask 3.1
  • Database: MongoDB 7.0 (Community Edition)
  • Frontend: Bootstrap 5.3, Jinja2, Vanilla JS
  • Intelligence: OpenRouter API (Gemini 2.0 Flash)

2. Database Schema

The system uses 8 primary collections in the flask_todo_db database:

CollectionPurpose
usersCredentials and Roles (Owner, Cashier)
itemsInventory Master List & Metrics
purchaseSales Ledger History
categoriesDynamic grouping for items
notesSystem Bulletin Board data
inventory_logRaw In/Out transaction history
system_logsAudit trail for all user actions
subscriptionsWeb Push device tokens

3. Core Algorithms

Sales Velocity (Inventory Health)

Identifies items as Cold Stock if last log date > 30 days. Categorizes as Sporadic if total volume < 5 units.

Profit Calculation
Profit = |Retail Price - Cost Price| * Quantity Sold
Inventory Value
Inv Value = (Current Stock + Total Sold) * Cost Price

4. PWA & Notifications

The system uses Web Push (VAPID) protocol for system-level notifications on Android and iOS.

Security Requirement

Push notifications and Service Workers strictly require HTTPS. Plain IP addresses will block these features.

Version: Service Worker v2.5
Filter: Auto-blocks SUBSCRIBE/UNSUBSCRIBE spam.

5. API Reference

GET /system-info

Returns live CPU, RAM, and Disk metrics.

GET /latest-log

Returns the single most recent activity for the toast system.

POST /subscribe

Registers a device's push subscription token.
